# V1 History (shipped 2026-06 / archived 2026-07-06)

Estudio V1: a single-user Spanish/English vocabulary + grammar SRS study app, built by an autonomous orchestrator loop over 178 iterations (2026-06-10 → 2026-06-14, plus prod data ops). One Node/Express/TypeScript process serves the REST API and the built React (Vite) app on one port, backed by one SQLite file (`data/app.db`, better-sqlite3, WAL). Release-done was declared at iteration 172/173; iterations 174-178 were live-data operations (Mochi import, KJV ingestion) driven by owner feedback.

This document is the compact inherited memory of V1. The full archives are `archive/v1/{GOAL,TASKS,DECISIONS,QUESTIONS,FEEDBACK}.md`.

## What shipped — feature inventory

- **Ingest** (`/ingest`, desktop-only by owner waiver) — PDF workbook scans (per-page vision extraction + vocab/grammar page classification), pasted text, Project Gutenberg books by ID/URL (local stopword pre-pass → batched LLM classification with an archaic-aware rubric), Mochi `.mochi` ZIP import (Transit-JSON parser), lesson-audio upload. `server/src/routes/sources*.ts`, `server/src/jobs/{pdfIngestion,textIngestion,gutenbergIngestion,gutenbergPrepass,mochiImport,lessonAudioIngestion}.ts`, `server/src/pdf/`.
- **Triage** (`/triage?source=N`) — know/learn/skip each extraction candidate, batch "Keep N" confirm materializes words with confirm-time lemma dedupe surfaced for human merge/keep. `server/src/routes/triage.ts`, `server/src/db/triage-queries.ts`, `web/src/screens/Triage.tsx`.
- **Library** (`/library`) — word CRUD, accent-insensitive search, ES/EN/All filter + pagination, manual add with auto-filled definition, delete (history retained), "I forgot this" demote, provenance. `server/src/routes/words.ts`, `web/src/screens/Library.tsx`, `web/src/components/Pagination.tsx`.
- **Review / SRS** (`/review`) — in-house SM-2; due queue with new-card promotion; MC default, flip-card fallback, cached cloze mix-in, opt-in Mochi-style yes/no format (settings-backed); instant grading. `server/src/srs/sm2.ts`, `server/src/routes/srs.ts`, `web/src/screens/Review.tsx`.
- **Quiz** (`/quiz`) — configurable def_match/cloze/mixed quizzes, client-side instant MC grading with async persist, cached questions reused before regeneration, explanations generated with the question, misses write review_log + pull the card due. `server/src/jobs/quizGen.ts`, `server/src/routes/quiz.ts`.
- **Grammar + Lessons** (`/grammar`, per-topic lessons) — LLM-seeded curriculum (button-triggered on /grammar; categories → topics), mastery EMA from attempts, generated lessons (explanation/examples + quiz_question practice) with MC/fill-in/conjugation/free-text answering and LLM grading ("Partly right" tier, grounded on the user's answer), page→topic curriculum links from PDFs. `server/src/jobs/{grammarSeed,lessonGen}.ts`, `server/src/routes/grammar.ts`.
- **Lesson recordings** (`/lessons`) — upload a real ~1 hr recording → ffmpeg split of >24 MB audio → Whisper transcription → `lesson_analysis` LLM mining of flagged words (→ triage), corrections, struggle sentences, topics covered. Proven end-to-end on the owner's real 24.8 MB recording ($0.41). `server/src/transcription/`, `server/src/jobs/lessonAudioIngestion.ts`, `server/src/routes/lessons.ts`.
- **Ask chatbot** (`/ask`) — persistent page-context-seeded chat threads, server-side tools (add_word_to_deck / lookup_word / get_page_context) with inline mutation confirmation, voice questions via MediaRecorder → transcription, single-click thread delete. `server/src/routes/chat.ts`, `prompts/chat.md`.
- **Suggestions** (`/suggestions`) — one-at-a-time word/grammar-topic proposals calibrated on known words; UNIQUE normalized key so nothing (including skips) is ever re-suggested. `server/src/routes/suggestions.ts`.
- **Quick add** — persistent "+" QuickAddModal (phone nav + desktop header) with mic dictation (`POST /api/transcribe`), plus TappableText: tap any word in app-rendered content (definitions, lesson examples, chat replies) to pre-fill QuickAdd. `web/src/components/{QuickAddModal,TappableText}.tsx`, `server/src/routes/transcribe.ts`.
- **Placement** (`/placement`) — optional adaptive English vocab placement (band climb/descend) seeding the likely-known calibration baseline. `server/src/placement/adaptive.ts`, `server/src/routes/placement.ts`.
- **Progress** (`/progress`) — status counts, 14-day due forecast, last-20-session quiz accuracy, per-source coverage, grammar-mastery heatmap. `server/src/routes/progress.ts`, `server/src/db/progress-queries.ts`.
- **Home** (`/`) — HomeHero (word of the day), OverviewCard entry-point grid, activity band, `whatNext` guidance nudge (suppressed while cards are due or triage is pending). `server/src/routes/overview.ts`, `web/src/components/HomeNudge.tsx`.
- **Notes** (`/notes`) — per-answer self-notes (note → quiz_question), browsable per word/topic, fed as context into future generation. `server/src/routes/notes.ts`.
- **System** (`/system`) — LLM + transcription spend, job statuses, recent errors, DB backup status, manual backup, one-click JSON export (`GET /api/system/export`) + .db download, preferences (definition display, new-cards/day, review format). `server/src/routes/{system,settings}.ts`.
- **Docs** — `docs/README.md` (cold start, phone via LAN/Tailscale, where data lives, backup/restore), `docs/demo.md`, `docs/spa-routing-assessment.md`, `docs/validation-pdf-ingestion.md`, `TODO-LATER.md`.

## Architecture as built

- **Stack:** Node + Express + TypeScript, React + Vite + plain CSS, SQLite (better-sqlite3, WAL), no ORM, vitest. Monorepo workspaces: `/server`, `/web`, `/shared` (API payload types, imported as `@estudio/shared` — server/web typecheck against `shared/dist`), `/prompts` (one versioned markdown template per LLM task, loaded at call time — no inline prompts), `/docs`, `/data` (git-ignored runtime: `app.db`, `backups/`, `uploads/`, `books/`; configurable via `DATA_DIR`).
- **Data model** (see `ARCHITECTURE.md` for the authoritative spec): `deck` (exactly 2 rows: es, en) · `source` (type pdf|text|lesson_audio|voice_question|gutenberg|mochi|manual|chat|suggestion; language es|en since migration 005; duration_minutes since 004) · `source_page` (per-page status/retry + page→grammar_topic link) · `extraction_item` (triage state; undecided candidates never become words) · `word` (status new|learning|mature|known|suspended; indexed `term_normalized`/`lemma_normalized`; definition_origin llm|owner; prompt_version) · `card_state` (one per learning word) · `review_log` (append-only) · `grammar_category`/`grammar_topic` (mastery REAL) · `lesson` (explanation/examples JSON only; cached forever) · `quiz_question` (cached, flagged-not-deleted; explanation generated with the question) · `quiz_attempt` · `note` · `lesson_insight` · `chat_thread`/`chat_message` · `suggestion` · `job` · `llm_call`/`transcription_call` (every call logged with cost) · `error_log` (capped ~1000) · `setting` (JSON KV). Migrations `server/src/db/migrations/001_init.sql`…`005_source_language.sql`, in-house runner, pre-migration auto-backup.
- **Seams:** `LlmProvider` (`server/src/llm/` — `complete()`/`vision()`, normalized usage/cost; one adapter per provider, `anthropic.ts` ships; per-task model defaults in `service.ts` `TASK_DEFAULTS`, overridable via `setting`) · `TranscriptionProvider` (`server/src/transcription/` — OpenAI Whisper adapter `openai.ts`, `ffmpegSplit.ts` with injectable `SplitAudio` seam) · persistent job queue (`server/src/jobs/queue.ts` + `handlers.ts` — `job` table polled in-process; >2s work is always a job; per-chunk resumable; `running`→`queued` on boot) · queries live in `server/src/db/*-queries.ts` (snake_case↔camelCase mapping happens there) · route modules registered in `app.ts` via `register<X>Routes(app, db, llm?, transcription?)`.
- **Validation scripts** (opt-in, not in check.sh): `server/src/scripts/{validate-ingestion,validate-lesson-audio,validate-gutenberg,run-kjv-prod}.ts`.
- **Web:** no router library — `App.tsx` switches on `window.location.pathname`; every nav is a full page reload (MPA-style; assessed, not converted). Component library in `web/src/components/`, tokens in `web/src/styles/tokens.css` materialized from `design/tokens.md`. `design/` is the design contract (INDEX identity D0/D1, tokens, 15 screen specs, components, interaction) — human-seeded, single-writer (orchestrator).

## Key decisions that still bind

1. **SM-2 in-house** (`server/src/srs/sm2.ts`, pure functions, no SRS library). Grades `fail|good|easy` ≈ SM-2 2/4/5; mature at interval ≥ 21 days; manual demotion = interval reset + ease −0.15, floor 1.3.
2. **`review_log` is append-only** — no code path updates or deletes rows; the schedule must be recomputable from the log alone.
3. **Word lifecycle:** triage "learn" → status `new`, NO card_state; the review queue builder promotes up to new_cards_per_day at session start, creating card_state and setting `learning`. Imports must insert `new` (a `learning` word without card_state is orphaned — the Mochi B1 bug).
4. **Word uniqueness = UNIQUE(term, language) exact-match only** — never on normalized forms (`más` ≠ `mas`). Lemma dedupe is a triage-surfaced check for a human decision, never a constraint. Extra senses append to definitions, never a second row.
5. Accent-insensitive search/dedupe via indexed `term_normalized`/`lemma_normalized` (lowercase + accent-strip written at write time); stored text keeps accents.
6. **IDs:** INTEGER PRIMARY KEY AUTOINCREMENT, no UUIDs. **Timestamps:** TEXT ISO-8601 UTC. snake_case tables, camelCase API JSON, mapping at the query layer. Errors: `{error:{message,code}}` with real status codes, never 200-with-error.
7. Hard-delete words (review_log orphans kept via ON DELETE SET NULL); migrations are numbered, additive-only, with automatic pre-migration backup; one schema change in flight, ever.
8. **Cache-first generation:** every LLM artifact (definition, lesson, question, explanation, curriculum, suggestion) is persisted and re-served before any regeneration; regeneration is a new row, old kept; flagged questions excluded from serving, never deleted.
9. **Provider isolation:** all calls through the adapter seams; no provider-specific types outside `llm/`/`transcription/`; per-task model config, never hardcoded at call sites; prompts only in `/prompts/*.md`; every call (success and failure) logged with cost.
10. **Model routing as tuned:** quality-critical tasks default `claude-opus-4-8` via the `FABLE_REPLACEMENT` constant (Fable disabled by government directive 2026-06-13; revert = one constant, sites tagged `FABLE-DISABLED`); `gutenberg_extraction` pinned to sonnet (owner cost feedback); `chat` → haiku; `suggestion_select` → sonnet; `quiz_grading` → sonnet/low.
11. `gutenberg_extraction` needs `maxTokens: 16384` — the 8192 default truncated 200-candidate batch JSON mid-string (found by the live KJV run). Cost estimates must err HIGH (constants calibrated to the real $7.41 run) so the >$5 in-app confirm gate fires reliably.
12. **Gutenberg pre-pass** (`gutenbergPrepass.ts`) is token-cost-only (stopwords/proper nouns); the semantic keep/drop is the LLM rubric (archaic-aware: drop thee/thou/unto noise, keep firmament/concupiscence-class words).
13. **Paid/large operations against live data require owner confirmation** ("dev is dev, production asks first"); workers write throwaway worktree DBs, so live-`data/app.db` data ops are run directly by the orchestrator/owner, never a worker.
14. §16 no-silent-loss: malformed import rows are counted and reported, never dropped silently; mutations attempt the write before flipping status (e.g. suggestion add) and surface failures.
15. Quiz misses write review_log rows and pull `card_state.due_at` to now; `quiz_attempt.style` records what actually ran (`mixed` allowed, never a fabricated single style); lesson quiz questions live only in `quiz_question`.
16. Suggestions: UNIQUE(item_type, normalized_key) — nothing re-suggested, skips included; deck exclusion matches lemma OR term normalized.
17. **Design contract:** plain CSS only; every visual value references a token custom property (no raw values where a token exists); no Tailwind/CSS-in-JS/UI framework/analytics; ≥44px tap targets; identity D0/D1 is human-owned and stays.
18. **Tests:** vitest single runner, colocated `*.test.ts(x)`; `check.sh` = shared build + server/web typecheck + builds + `npx vitest run` — the integration gate for all work; workers may never modify it.
19. `/ingest` is desktop-only — the GOAL §15 phone-ingest criterion was formally owner-waived (matchMedia 639px guard, not user-agent). *(V2 note: revisit under the Phase-1 polish bar.)*
20. `word-select-to-add` shipped REDUCED by owner-accepted decision: tappable words in app-rendered content only, not OS-level text selection (full version in TODO-LATER).
21. Secrets in `.env` only; the browser never calls a provider API; the server never serves a key; all runtime disk state under `DATA_DIR`.
22. Transcription = OpenAI Whisper (GOAL-pre-approved paid dependency, $0.006/min); >24 MB audio ffmpeg-split at 92% of the size cap with a bounded re-split retry on VBR spikes.
23. Grammar curriculum is seeded by an explicit button on `/grammar` (lessons hang off topics, not vocab); "seen in lessons" and the practice queue are derived at read time — no stored counters to drift.

## Known weaknesses / deferred items

- **TODO-LATER.md (owner-acknowledged v1 cuts):** TTS/audio pronunciation playback; SMS/email word-capture inbox; combined vocab+grammar quizzes; subjects beyond Spanish/English; full (OS-level) word-select-to-add.
- **suggestion-streaming** — the only backlog item never built: streaming on the `LlmProvider` seam → SSE → progressive render in Suggestions. Deliberately deferred since iter 146 (needs a focused seam design; marginal payoff on short structured output; would pay off more on chat/explanations).
- **No SPA routing** — `App.tsx` pathname-switch, every nav is a full document reload. Scoped in `docs/spa-routing-assessment.md` but conversion was never queued. *(Directly relevant to the V2 Phase-1 "mobile feel" bar.)*
- **Single-user by construction** — no auth, no user table, one SQLite file, two fixed decks, one `setting` KV; the whole data model and every query assumes one owner. This is the central V1→V2 gap (GOAL v3 Phase 2).
- **KJV 36/46 chunks unprocessed** (see above) — the English deck's book coverage indicator reflects only 10 chunks.
- Cost/robustness lessons already fixed but worth knowing: LLM batch truncation (fixed via maxTokens), estimate undershoot (fixed with headroom), flaky RTL timeouts in Triage tests (fixed with explicit timeouts) — the pattern is "live runs find what mocked tests can't"; V1's §14 proofs (real PDF, real 1-hr recording, real full KJV) were all separate orchestrator-run validations.
- Long-lived Node processes suffered from firewall/CDN-IP rotation in the sandboxed container (KJV resume stalls); resumable per-chunk jobs were the mitigation. Only relevant if V2 keeps the containerized ingestion pattern.
